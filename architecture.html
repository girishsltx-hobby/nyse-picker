System Design

Full-stack real-time NYSE intraday dashboard. Python/FastAPI backend with a 5-minute scheduler, 33-rule AI evaluator (RSI, EMA, VWAP, Volume, Confluence), 5 composite alert detectors (POWER_TREND, STRUCTURE_BREAK, VWAP_RESET, EXHAUSTION_REVERSAL, POC_MAGNET) with batch display and manual refresh, optional LLM commentary (Gemini / OpenAI / Claude / Ollama), SQLite persistence, and a React/Zustand frontend connected via WebSocket.

AI Evaluation Engine
Rule-based evaluator — ai/evaluator.py
Deterministic weighted-vote system backed by optional LLM commentary. Every rule that fires is named, weighted, and explained in the UI. v3 adds RSI momentum, EMA cross-imminence, and a confluence synthesiser (9-signal alignment score).
1

Receive snapshot dict

run_evaluation(ticker, df, current_price, snapshot) called after indicators are computed. snapshot contains all 30+ fields from the indicators modules.

2

Evaluate each rule

Each of the 27 rules tests one condition on the snapshot. Returns True if triggered, False otherwise. Rules have a direction (UP or DOWN) and a float weight.

3

Sum weighted votes

UP-triggered rules contribute +weight to score; DOWN rules contribute -weight. Final score determines direction. confidence = |score| / total_possible_weight.

4

LLM Commentary (optional)

If LLM_PROVIDER is set, ai/llm_commentary.py calls Gemini Flash / GPT-4o-mini / Claude Haiku / Ollama to generate a 2-sentence plain-English rationale. Rate-limited to 1 call per 2.2 s. Only fires when prediction direction or confidence shifts ≥10% — typically 2–5 calls/hour. Result stored in notes field.

5

Emit PredictionRow

Stores prediction (UP/DOWN/NEUTRAL), confidence (0–1), rules_triggered as JSON array, and optional notes (LLM commentary) into the predictions table.

6

Outcome resolution

resolve_outcomes() runs post-cycle. For predictions older than 30 min, checks if the price moved in the predicted direction and marks outcome as WIN/LOSS in the DB. Feeds future ML retrain.

LLM Commentary
Plain-English trade rationale — ai/llm_commentary.py
Optional layer that calls an LLM after each rule-engine evaluation to produce a 2-sentence intraday rationale. Disabled by default; enabled by setting LLM_PROVIDER env var.
1

Change detection

Compares new prediction to the last emitted prediction for the same ticker. Skips the LLM call unless direction flipped (UP↔DOWN/NEUTRAL) or confidence shifted ≥10 pp. Keeps calls to ~2–5/hour across 10 tickers.

2

Rate limiter

Global asyncio.Lock enforces ≥2.2 s between any two LLM calls regardless of ticker. Prevents burst rate-limit errors on free-tier API keys (Gemini free = 30 RPM).

3

Provider dispatch

Switches on LLM_PROVIDER env var to one of four async call helpers:

ProviderEnv var valueDefault modelBase URL
Google Geminigeminigemini-2.0-flash-litegenerativelanguage.googleapis.com
OpenAIopenaigpt-4o-miniapi.openai.com

Ollama (local)ollamallama3.1localhost:11434 (configurable)
4

Silent fail

Any error (network, auth, quota) is caught, logged as a warning, and returns None. The prediction is still persisted and broadcast; only the notes field is absent. No user-visible crash.

5

UI display

When pred.notes is non-null, TickerDetail renders a purple 🤖 AI COMMENTARY block above the triggered-rules list. Commentary is plain prose summarising direction, confidence, and key signals.
